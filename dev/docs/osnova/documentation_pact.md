# Пакт о документации ОСНОВА ИТ

## 1. Назначение
Документация является частью продукта и инфраструктуры компании ОСНОВА ИТ.
Отсутствие документации приравнивается к отсутствию функциональности.

## 2. Цели
- Сохранение коллективного знания
- Масштабируемость команд и решений
- Снижение bus-factor
- Подготовка к AI / Vector-based поиску
- Прозрачность для разработчиков, заказчиков и пользователей

## 3. Основные принципы
1. Docs-as-Code — документация живёт в Git
2. Single Source of Truth
3. Актуальность важнее полноты
4. Структура важнее формы
5. Документация пишется для человека, читается машиной

## 4. Обязательства
- Любая фича считается незавершённой без документации
- Архитектурные решения фиксируются письменно
- Интеграции с легаси документируются отдельно
- Документация обновляется в том же PR, что и код
- Коммиты оформляются по единому стандарту, чтобы история была читаемой и машинно-обрабатываемой
- Работа ведется через GitHub с обязательными проверками и ревью

## 5. Типы документации
- Архитектурная
- Техническая
- Интеграционная
- Пользовательская
- Операционная
- AI-готовая (семантически структурированная)

## 6. AI-Ready принцип
Документация:
- хранится в Markdown
- имеет чёткую структуру
- не содержит «воды»
- пригодна для векторизации и RAG

## 7. Ответственность
Каждый автор кода — автор документации.
Каждый тимлид — хранитель актуальности.
Каждый автор коммита — ответственный за корректное оформление сообщения и связь с задачей.

## 8. Кредо
«Если это не задокументировано — этого не существует»
